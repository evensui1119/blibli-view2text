## 🖥️ 二、环境准备：三系统适配策略（Windows / macOS / Linux）

**Jen:**  
我们以一台刚装好纯净 Windows 系统的电脑为起点——没有开发环境、没有依赖、没有配置。这是最“差”的开局，但你的实际环境，大概率比它强得多。

**Mark:**  
虽然本教程以 Windows 为主，但要特别说明：macOS 和 Linux 用户反而更简单——它们原生兼容 Cloud Code 的命令体系。关键操作步骤，我会同步标注 macOS/Linux 命令。

**John:**  
第一步：Windows 用户需安装 **Git**（macOS/Linux 用户跳过）。  
Git 在这里不是为了代码管理，而是充当“命令翻译器”——因为 Cloud Code 原生适配 Unix-like 环境，Git for Windows 提供了必要的 POSIX 兼容层（如 bash、ssh、curl 等），让 Cloud Code 能在 Windows 上真正“原生运行”。

> ✅ 安装提示：官网下载慢？可直接使用作者提供的加速安装包；全程默认设置，“狂点下一步”即可——确实有十几步，但无需任何自定义。

## ⬇️ 三、安装 Cloud Code：绕过区域限制的可靠方案

**Jen:**  
打开 PowerShell（右键「开始菜单」→「终端（管理员）」），输入官方安装命令——但注意：Cloud Code 官方服务目前屏蔽中国大陆 IP，直接运行会报错。

**Mark:**  
别慌。我们改用作者优化后的镜像命令：  
```powershell
irm https://cdn.jsdelivr.net/gh/dahei-ai/cloudcode-installer@main/install.ps1 | iex
```  
这个脚本本质是官方安装逻辑的完整复刻，唯一增强是添加了实时下载进度条。即使无法连接官方服务器，只要能访问 jsDelivr（国内通常可达），就能稳定下载那个约 200MB 的核心安装包。

**John:**  
安装成功后，你会看到清晰提示：✅ 成功安装 + 当前版本号 + 安装路径（默认为 `C:\Users\{用户名}\.cloud`）。  
但此时还不能直接输入 `cloud` 启动——就像你把软件装进了 C 盘深处，却没给系统“指路”。

## 🌐 四、配置环境变量：让 `cloud` 命令全局生效

**Jen:**  
类比图形界面：双击运行程序，靠的是桌面快捷方式；而在命令行中，“快捷方式”就是**环境变量中的 PATH**。

**Mark:**  
操作路径很直观：  
1. 开始菜单搜索「环境变量」→ 点击「编辑系统环境变量」  
2. 点「环境变量」按钮 → 在「系统变量」区域找到 `Path` → 双击编辑  
3. 点「新建」→ 粘贴刚才安装提示中的完整路径（例如 `C:\Users\Alice\.cloud\bin`）→ 保存  

**John:**  
⚠️ 关键提醒：修改后务必关闭当前 PowerShell，重新打开一个——否则系统不会加载新 PATH。此时输入 `cloud`，即可立即启动，无需长路径复制粘贴。

## 🔑 五、登录困境破局：用 CC Switch 自由切换 AI 模型

**Jen:**  
启动 Cloud Code 后，第一道坎来了：它默认引导你登录官方账号、绑定订阅——但多数人既无会员，也无官方 API Key。

**Mark:**  
这里必须厘清一个核心概念：**Cloud Code 是框架，不是模型本身。**  
它就像一辆高性能汽车的底盘和驾驶舱，而引擎（AI 模型）完全可以自主更换——你可以用通义千问、Kimi、GLM、DeepSeek，甚至本地部署的 Ollama 模型。

**John:**  
解决方案是：安装 **CC Switch**（轻量级模型路由工具）。  
- 下载安装后，它会在系统托盘提供图形化界面；  
- 支持添加任意符合 OpenAI API 标准的渠道（含国内大厂官方接口、合规中转站等）；  
- 作者已整理近 200 个可用 API 地址（含密钥生成指引），新手可直接选用；  

**Jen（演示）：**  
以智谱 GLM-4-Flash 为例：  
1. 访问 [zhipu.ai](https://www.zhipu.ai) → 控制台创建 API Key  
2. 在 CC Switch 中新增渠道：填入 Key、指定模型名 `glm-4-flash`、保存并启用  
3. 重启 Cloud Code —— 登录页消失，左下角直接显示 `glm-4-flash`，对话即刻开始。

## 🛠️ 六、真实工作流：在终端中指挥 AI 操控你的电脑

**Mark:**  
很多用户困惑：“为什么非要在黑乎乎的终端里用？”  
答案很本质：**图形界面对 AI 是‘盲区’，而命令行是它的母语。**

**John:**  
想象一下：你想让 AI 帮你格式化一块异常的 U 盘。  
- 图形方式：AI 需截图→OCR识别按钮→模拟鼠标移动→点击→再截图→再判断……Token 暴涨，响应迟缓，极易出错。  
- 命令行方式：你只需说：“帮我识别并安全格式化挂载为 `/dev/sdb` 的U盘”，AI 即刻输出精准 `fdisk` + `mkfs` 命令，你确认后一键执行。

**Jen:**  
这就是 Cloud Code 的革命性所在——它把 AI 变成了你电脑的「超级命令行助手」：  
✅ `ls` / `cat` / `cp` / `git clone`… 所有文件操作，AI 都能理解、规划、生成、验证、执行；  
✅ 写代码？自动补全、调试、生成测试、提交 PR；  
✅ 查故障？分析日志、检测进程、重置网络；  
✅ 下项目？`cloud git clone https://github.com/xxx/yyy`，自动处理依赖与环境；  

**Mark:**  
它不替代你思考，而是把你脑海中的意图，精准翻译成计算机可执行的原子指令——这才是人机协作的终极效率。

## 📚 七、进阶技巧：上下文管理与会话延续

**Jen:**  
Cloud Code 默认保留多轮对话上下文。以 GLM-4-Flash 为例，上下文窗口达 200K tokens，足够支撑长篇技术讨论或复杂项目推进。

**John:**  
但要注意：当上下文趋近上限时，系统会自动压缩历史（lossy compression），早期细节可能失真。  
- 手动压缩命令：`/compact`（慎用，不可逆）；  
- 中断后回归：进入原工作目录，运行 `cloud --resume`，无缝接续上次对话；  
- 工作目录即项目根目录——右键文件夹 → 「在终端中打开」，是最自然的启动姿势。
