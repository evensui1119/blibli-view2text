# find-job

求职辅助：双路径（公司调研 / 岗位搜索）+ 单 JD 评估 + 三维评分卡 + 结构化落盘。

## 何时使用

- 用户给出公司名，希望了解业务、组织、AI 浓度、稳定性、口碑
- 用户给出搜索条件（城市 / 方向 / 关键词），希望找出匹配岗位
- 用户粘贴 JD 文本或招聘链接，希望评估匹配度
- 触发关键词：「调研 / 了解 / 看看 XX 公司」「找 XX 岗」「这个岗位怎么样」「我适合 XX 吗」「帮我找工作」

## 不适用场景（提示用户走他途）

- 简历优化、面试题准备：超出本 skill 范围
- 薪资谈判策略：超出本 skill 范围
- 跨多公司的横向对比矩阵：可在生成多份单公司报告后由用户另起任务做汇总

## 工作流（5 步）

### Step 1：加载用户画像（每次必做）

读取 `workfinder/profile.md`：

- **存在** → 解析其中的「目标方向 / 硬条件 / 忌讳项」作为后续匹配基准
- **不存在** → 必须先停下询问用户：
  > 未找到 `workfinder/profile.md`，它是匹配评分的基础。是否现在用模板生成（你之后填充内容）？

  得到肯定回答后，用「附录 A: profile.md 模板」创建文件，再继续。否则中止，提示用户补全画像后再调用。

### Step 2：意图识别

| 用户输入特征 | 走的路径 |
|---|---|
| 仅给出公司名 | 【公司调研】 |
| 含搜索条件（城市 + 方向 + 角色） | 【岗位搜索】 |
| 完整 JD 文本 / 招聘详情 URL | 【单 JD 评估】 |
| 含糊（"找工作 / 看看机会"） | 反问澄清意图，不要猜 |

### Step 3：数据获取（按优先级择一或组合）

1. **用户已粘贴的 JD / URL** → 直接使用，不再二次搜索
2. **WebSearch + WebFetch** → 公开信息：官网、招聘公告、媒体报道、年报、企查查类站点
3. **browser-use MCP** → Boss 直聘、LinkedIn 等需要登录或动态渲染的站点；若需登录，提示用户在浏览器里完成后继续

⚠️ **数据来源标注硬规则**：报告里每条事实必须在末尾用 `[来源: URL 或 "用户粘贴"]` 标注。无来源 = 不写。信息缺失时显式写「未查到」，不得编造。

### Step 4：三维评分卡（每条必出）

| 维度 | 评估子项 | 数据从哪来 |
|---|---|---|
| **硬条件** | 城市 / 薪资范围 / 是否 leader 岗 / 远程支持 / 是否 OD-外包 | 与 `profile.md` 硬性匹配 |
| **AI 技术深度** | 是否真做 Agent/LLM、商业化阶段、AI 团队规模、技术决策权、技术博客与开源 | 招聘文案的技术名词密度 + 公司技术输出（博客 / GitHub / 公开演讲） |
| **公司稳定性** | 上市/融资阶段、近 1 年组织变动、外部口碑、是否大规模裁员 | 企查查、脉脉、媒体报道，标注信息源 |

每维 0–10 分，总分 30 分。判定阈值：

- **≥ 24**：推荐进一步动作
- **18 – 23**：观望，列出待补信息
- **< 18**：排除，写明排除主因

### Step 5：落盘

| 路径 | 输出位置 | 文件命名 |
|---|---|---|
| 公司调研 | `workfinder/companies/` | `{公司名}.md` |
| 单 JD 评估 | `workfinder/jobs/` | `{公司名}-{岗位名}-{YYYYMMDD}.md` |
| 批量搜索筛选 | `workfinder/reports/` | `{主题描述}-{YYYYMMDD}.md` |

- 目录不存在自动创建
- 文件已存在 → 询问用户：覆盖 / 追加新章节 / 新建副本（加 `-v2` 后缀）
- 落盘后用 markdown 链接形式回报路径

## 关键约束

1. **不编造**：所有事实必须有来源；信息缺失时显式写「未查到」
2. **不过度调研**：单次任务控制在 3–5 个数据源，避免无限延展
3. **保护隐私**：`workfinder/` 全部产物默认不应入库；落盘前若发现 `.gitignore` 未排除 `workfinder/`，提示用户考虑加入
4. **复用现有清单**：`workfinder/` 根目录的「央国企业.md / 新能源与上市公司.md / 民营企业-互联网大厂.md / 科研院所.md / 公司评价.md」是已有池子；调研新公司时先查现有清单避免重复，调研完后可回填到对应清单

## 附录 A：`profile.md` 模板

skill 首次运行且文件缺失时，用以下内容创建：

```markdown
# 个人画像

> 本文件用于 find-job skill 的匹配评分。修改后下次调用自动生效。

## 当前定位
{例：8 年研发，3 年 AI Agent / LLM 应用方向，团队 leader}

## 技术栈
- 核心：{LLM 应用框架 / Agent 编排 / RAG / ...}
- 熟悉：{后端语言 / 中间件 / ...}
- 加分：{开源贡献 / 论文 / 公开演讲 / ...}

## 业务经验
- {电商导购 / 推荐 / 搜索 / ...}
- {团队管理：x 人规模 / x 年}

## 目标方向
- 一档（最想去）：…
- 二档（可考虑）：…
- 排除（不去）：…

## 硬条件
- 城市：{杭州/北京/上海/远程}
- 薪资底线：{年包 W}
- 远程：{接受 / 不接受 / 部分}
- leader 岗：{必须 / 可放宽 / 不要求}
- 入职时间：{即时 / x 个月内}

## 忌讳项
- {例：纯外包 OD / 996 / 早期初创无营收 / ...}
```

## 附录 B：与现有 workfinder/ 资产的协同

| 已有文件 | 协同方式 |
|---|---|
| `workfinder/README.md` | 索引文件，新调研产物落盘后建议在此追加目录链接 |
