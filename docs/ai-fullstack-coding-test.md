# AI 全栈开发笔试题

## 技术要求

基于 **Node.js / TypeScript** 开发，代码需部署到 **Cloudflare Worker**，并提供可以公开访问的网址。

> 欢迎使用 AI 生成代码，但模块拆分要清晰，代码要简洁、优雅，忌臃肿；要能体现你的技术审美、工程判断和产品品味。

---

## 任务说明

思考、设计、尽可能复现演示效果。

---

## 基本要求

1. 提供一个网页，用户可以输入一个有字幕的 YouTube 视频链接。
   - 建议额外硬编码一份字幕结果以避免 YouTube 字幕获取不稳定。

2. 调用 **Gemini AI Studio 免费 API**，基于 YouTube 字幕生成一篇**中文视频对话内容文章**。
   - 文章需有清晰排版，并在网页中渲染为 HTML。

3. 主文章生成**必须使用流式输出**：生成一点输出一点，并实时展示在网页上。

### 遇到 YouTube 验证码怎么解决？

如果 YouTube 提取字幕遇到验证码，可以考虑用 [webshare.io](https://webshare.io) 的代理（免费账号有 10 个代理，无需信用卡，用谷歌登录即可）。

Cloudflare Worker 的 `fetch` 不支持配置代理，你可以用 **Cloudflare Worker 的 TCP SOCKET** 发起请求。

## 提升要求

### 1. 自然语言生成要求

页面还需支持输入一段**自然语言生成要求**（使用时可选）。生成内容应能体现用户要求中的约束（不一定都覆盖，但不会超过这个范围）：

| 维度 | 说明 |
|------|------|
| 任务类型 | 用户指定的内容类型 |
| 输出风格 | 文章的写作风格 |
| 目标受众 | 面向的读者群体 |
| 约束条件 | 其他限制性要求 |

### 2. 5W1H 总结

主文章生成结果需要**按章节组织**。每个章节需提供「5W1H 总结」能力：

- 用户点击章节标题旁 **[5W1H]** 按钮后，页面展示该章节的 Who / What / When / Where / Why / How 总结。
- 总结需结合**整篇视频内容**与**当前章节上下文**，并以结构化数据返回、固定格式渲染。
- 章节「5W1H 总结」请求**不得由前端重新提交整篇文章内容**；系统应基于服务端保存的本次生成上下文完成总结。
